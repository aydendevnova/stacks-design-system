# Web3 Patterns

Design patterns specifically crafted for Web3 applications on Stacks. These patterns address the unique UX challenges of blockchain interactions.

Building decentralized applications requires handling complex interactions like wallet connections, transaction signing, and async blockchain operations. These patterns provide tested solutions for common Web3 UX challenges.

## Wallet Connection

| Pattern | Description |
|---------|-------------|
| [Connection Flow](./wallet-connect) | Connect wallet button, wallet selection modal, and connected state |
| Account Display | Displaying connected addresses and account information |

## Transactions

| Pattern | Description |
|---------|-------------|
| [Transaction States](./transactions/states) | Pending, confirmed, and failed transaction UI patterns |
| Fee Display | Showing network fees and gas estimates to users |
| Confirmation Modals | Transaction review and confirmation dialogs |

## DeFi Patterns

| Pattern | Description |
|---------|-------------|
| Staking Flows | Stake, unstake, and rewards claim interfaces |
| Swap Interface | Token swap UI with price impact and slippage |
